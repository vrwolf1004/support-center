# Help Me Hide 개인정보 처리방침

최종 수정일: 2026-04-23

## 1. 수집하는 정보

Help Me Hide는 서비스 제공을 위해 다음 정보를 처리합니다.

| 구분 | 항목 | 목적 | 저장 위치 |
|------|------|------|----------|
| 사용자 입력 | PIN 번호 | 암호화 키 생성 | 사용자 기기 |
| 사용자 설정 | 암호화 모드 (AES-GCM, Simple) | 암호화 방식 선택 | 사용자 기기 |
| 사용자 설정 | 마커 형식 (HTML 주석, 괄호) | 마커 표시 방식 | 사용자 기기 |

- **개인 식별 정보(이름, 이메일, 계정 정보 등)는 수집하지 않습니다.**
- **암호화된 텍스트는 사용자가 직접 관리하며, 확장이 보관하지 않습니다.**
- **PIN 번호는 로컬 저장소에만 저장되며, 외부로 전송되지 않습니다.**

## 2. 정보의 이용 목적

- PIN 기반 암호화/복호화 기능 제공
- 사용자 암호화 설정 저장
- 마커 형식 관리

## 3. 정보의 제3자 제공

수집된 정보는 원칙적으로 제3자에게 제공하지 않습니다.

**예외**:
- 법령에 따른 요청이 있는 경우
- 사용자의 명시적 동의가 있는 경우

## 4. 정보의 보관 및 파기

- **PIN 저장소**: Chrome Storage에 암호화되어 저장
- **설정**: Chrome Storage에 저장, 확장 삭제 시 함께 삭제
- **암호화된 텍스트**: 사용자 책임 (클립보드, 문서 등)
- **자동 파기**: 확장 프로그램 삭제 시 저장된 모든 PIN 및 설정 자동 삭제

## 5. 사용자의 권리

- **언제든지 확장 삭제**: 확장을 제거하면 저장된 PIN이 모두 삭제됩니다
- **PIN 관리**: 언제든지 PIN 변경 가능
- **암호화 방식 선택**: AES-GCM과 Simple 모드 중 선택
- **데이터 제어**: 암호화된 텍스트는 사용자가 직접 관리

## 6. 보안 주의사항

- **약한 보안**: 4자리 PIN은 매우 약한 보안입니다 (10,000가지 조합)
- **교육 목적**: 이 확장은 교육 및 개념 검증 목적으로 설계되었습니다
- **민감정보 금지**: 비밀번호, 금융정보 등 민감한 정보 저장 금지
- **전문 도구 사용**: 실제 강력한 보안이 필요하면 전문 암호화 도구 사용 권장

## 7. 기술 정보

**암호화 방식**:
- **AES-GCM**: PBKDF2 키 도출, 랜덤 IV, 128-bit 인증 태그
- **Simple XOR**: 빠른 처리 (보안 약함)

**저장 메커니즘**:
- Chrome Storage: PIN, 설정 저장
- 마커 형식: `<!--ENC:V1:salt:nonce:ciphertext-->` 또는 `[ENC:V1:...]`

**외부 통신**:
- 없음 (모든 처리가 로컬에서 수행)

## 8. 정책 변경

개인정보 처리방침은 서비스 개선을 위해 변경될 수 있습니다.
변경 시 확장의 업데이트 공지를 통해 안내됩니다.
